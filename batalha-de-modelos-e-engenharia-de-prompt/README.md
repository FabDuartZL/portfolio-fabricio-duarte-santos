# 🌙 NightHub: Batalha de Modelos & Engenharia de Prompt

## 📝 Descrição do Projeto
[cite_start]Este projeto apresenta um estudo comparativo de **Engenharia de Prompt** utilizando uma estrutura baseada em **XML** para a criação do **NightHub**[cite: 2, 7]. [cite_start]O NightHub é um sistema SaaS focado na gestão e divulgação de eventos, promoções e geolocalização para estabelecimentos como adegas e tabacarias[cite: 25, 31].

[cite_start]Desenvolvido como parte da disciplina de **Engenharia de Prompt e Aplicações em IA (2026)**, o experimento testou a capacidade de diferentes Grandes Modelos de Linguagem (LLMs) em gerar uma aplicação *Single Page* funcional com requisitos técnicos complexos, como sistemas de notificações push e painéis de publicação em tempo real[cite: 1, 25, 28].

## 🚀 Tecnologias e Metodologia
* [cite_start]**Linguagens Geradas:** HTML5 e CSS3 integrado[cite: 5].
* [cite_start]**Estruturação de Prompt:** Tags XML (`<tarefa>`, `<objetivo>`, `<diretrizes_design>`) para maior precisão[cite: 7].
* [cite_start]**Modelos Avaliados:** ChatGPT, Gemini, Claude, Qwen, DeepSeek, Grok e Maritaca[cite: 6].
* [cite_start]**Design:** Estética Dark com paleta em Preto, Roxo e Neon[cite: 7].

## 💡 Resultados e Aprendizados
* [cite_start]**Eficiência de Tokens:** Observou-se que modelos com maior contagem de tokens (como Qwen e Claude) entregaram resultados mais robustos e completos[cite: 18].
* [cite_start]**Compreensão Estrutural:** O **Grok** demonstrou a melhor compreensão da estrutura XML solicitada, atendendo rigorosamente aos requisitos de design e layout[cite: 14].
* [cite_start]**Prototipagem vs. Complexidade:** O **DeepSeek** foi identificado como ideal para prototipagem rápida [cite: 21][cite_start], enquanto o **Claude** mostrou-se superior para códigos complexos, especialmente com supervisão humana[cite: 22, 23].

## 🔧 Como Replicar
1. [cite_start]Utilize o prompt estruturado em XML disponível na documentação[cite: 7].
2. [cite_start]Insira as diretrizes específicas de negócio (SaaS NightHub)[cite: 25].
3. Execute em um dos modelos de IA citados para gerar o arquivo `index.html`.
4. [cite_start]Abra o arquivo gerado em qualquer navegador moderno para visualizar a interface responsiva[cite: 28].
